Philosophy

CMMC Advisor is an enabler, not a gatekeeper. When a compliant path exists, the skill maps it clearly. When no compliant option exists today, it identifies the gap, names who is working on it, and suggests interim measures. Answers that block or refuse without a path forward are failures.

Every factual claim traces to public sources listed in SOURCES.md. Interpretive guidance (cloud selection, SSP narrative drafting) does not require citation but must not contradict cited sources.

Three HTML views (different audiences)

Program dashboard

ISSM / security team. Full objective status, SPRS score, POA&M clocks, evidence freshness. Internal.

Trust center

Customers and partners. Deny-by-default public posture page. No gap counts or SPRS.

Executive brief

CFO / C-level. Top gaps by point value, budget hints, Marketplace/CIS solution recommendations. Internal only.

Typical workflow

  1. Copy templates/program-data.sample.yaml to your compliance repo as program-data.yaml.
  2. Fill organization, assessment path, per-objective conformity, narratives, evidence links, inheritance, POA&M.
  3. Run collectors or import GRC snapshots into evidence/.
  4. Regenerate: python3 scripts/generate_dashboard.py program-data.yaml -o dashboard.html
  5. Export SPRS: python3 scripts/export_sprs.py program-data.yaml -o exports/sprs.json --csv exports/sprs.csv

Evals & CI

The eval runner (evals/runner/) invokes Claude as the subject using the skill, runs deterministic voice/citation lint, then scores against rubrics.

pip install -r evals/runner/requirements.txt
python -m evals.runner.runner evals/scenarios/level-2-scoping-basic.yaml
python3 evals/runner/lint.py   # CI voice/citation lint on SKILL.md + references/

  • Domain practices, assessment objectives, SSP authoring
  • POA&M eligibility, GRC continuous monitoring, vendor treatment
  • Modern IT (M365 tenancy, AI dev tools, remote access, VDI/IGEL)

Known limits & honest gaps

  • Collector HTTP clients: Most cloud/SaaS and on-prem collectors are manifest + stub modules. Live API wiring is org-specific or delegated to vendor MCP / GRC inspector plugins. GCP live ConMon uses the external Meridian bridge.
  • GRC import does not certify: Snapshots add monitoring metadata and evidence links; conformity status requires human review.
  • FedRAMP 20x ≠ CMMC reciprocity: A FedRAMP 20x authorization does not substitute for CMMC Level 2 certification. Documented in solution-selection guidance.
  • Trust center vs executive brief: Never publish SPRS scores, gap counts, or POA&M details on the trust center.
  • Harness multi-agent specialists: Out of scope for this distribution (lives in author's separate harness repo).
  • Cursor Marketplace: Local plugin stub exists; marketplace submission is optional follow-on.
  • Scheduled sync: No built-in cron for GRC snapshots; orgs run import on their own schedule or CI.
  • Not legal advice: Guidance from public sources only; verify before SSP submission.
